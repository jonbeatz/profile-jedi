# Feature Specifications

One folder per major feature. Each folder contains:

- `plan.md` — the **What** and **Why** (requirements, strategy)
- `tasks.md` — actionable task list
- `notes.md` — learnings, edge cases, technical discoveries

## Usage

When starting a new major feature:

1. Copy `specs/_template/` → `specs/your-feature-name/`
2. Fill out `plan.md` first
3. Break down into `tasks.md`
4. Implement and log discoveries in `notes.md`
5. Summarize shipped work in `CHANGELOG.md`
